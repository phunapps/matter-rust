# connectedhomeip certificate test vectors

Second reference set for `matter-cert`'s byte-parity gate, generated by
the CSA C++ implementation's `chip-cert` tool (the matter.js set lives
one directory up). Regenerate with `cargo xtask capture-cert-chip`
(requires a built chip-cert; see the rustdoc in
`xtask/src/capture_cert_chip.rs`).

`<id>.bin` is the raw CHIP TLV certificate; `<id>.tbs.bin` is the
TBSCertificate slice of chip-cert's X.509 DER output — the exact bytes
the CA signed, which `MatterCertificate::to_x509_tbs_der()` must
reproduce byte-for-byte.

Keys are generated fresh per capture and are NOT committed; the tests
exercise whatever bytes are committed here.
