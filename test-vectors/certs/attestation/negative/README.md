# Synthetic negative-path attestation chain fixtures

8 self-contained (PAA, PAI, DAC) triples generated by
`scripts/gen-negative-fixtures.py`. Each fixture has exactly one
deliberate flaw that exercises a specific
`matter_commissioning::AttestationError` variant.

## Anchor

All fixtures are timestamped relative to `AT_UNIX = 1_800_000_000`
(2027-01-15T08:00:00Z). The integration test pins `at:
MatterTime::from_unix_secs(1_800_000_000)`.

## Fixtures

| Directory | Deliberate flaw | Expected variant |
|---|---|---|
| `expired-dac/` | DAC `notAfter` < `AT` | `TimeBoundsViolation` |
| `not-yet-valid-dac/` | DAC `notBefore` > `AT` | `TimeBoundsViolation` |
| `broken-dac-sig/` | Last 2 bytes of DAC signature XORed with 0x01 | `InvalidChain` |
| `broken-pai-sig/` | Last 2 bytes of PAI signature XORed with 0x01 | `InvalidChain` |
| `wrong-vid-dac/` | DAC subject VID=0xFFF2; PAI subject VID=0xFFF1 | `VidMismatch` |
| `untrusted-paa/` | Chain rooted in PAA-OTHER; trust store has PAA-FFF1 instead | `UntrustedRoot` |
| `dac-with-ca-bit/` | DAC `BasicConstraints.cA = true` (forbidden on leaf) | `BasicConstraintsViolation` |
| `wrong-eku/` | DAC EKU contains `id-kp-serverAuth` instead of `id-kp-clientAuth` | `InvalidChain` (webpki rejects with `RequiredEkuNotFound`) |

> Note: the M6.2 spec originally specified a `missing-eku` fixture
> ("DAC lacks `id-kp-clientAuth` EKU"). webpki (correctly, per RFC
> 5280 §4.2.1.12) treats an absent EKU extension as unconstrained
> — no required-EKU check fires. So this fixture instead has an
> EKU extension with the *wrong* OID, which webpki rejects with
> `RequiredEkuNotFound`.
