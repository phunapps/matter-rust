# M8.3 Runbook — Commission a real device via `MatterController`

Operator-gated validation for M8.3: commission a real Matter device through the
high-level `MatterController` API, confirm it is persisted, control it over an
operational session, and confirm a restart reconnects from the snapshot without
re-commissioning.

## Device
Tapo P110M (the device validated in M6.6 / M7.5). Factory-reset it before each run
(hold the button until it blinks; remove from the Tapo app if previously paired).

## Trust material (REQUIRED for a real device)
Real certified devices are attested against **production** PAA roots; the bundled
CSA test roots reject them (observed M7.5 — `PAA not in trust store`). Production
roots live in a connectedhomeip checkout:

- PAA roots dir: `/Users/hemanshubhojak/code/connectedhomeip/credentials/production/paa-root-certs`
- CD signing roots dir: `/Users/hemanshubhojak/code/connectedhomeip/credentials/production/cd-certs`

Build the controller with:

```rust
let controller = MatterController::builder(store)
    .attestation_trust(AttestationTrust::from_dirs(
        Path::new(".../credentials/production/paa-root-certs"),
        Path::new(".../credentials/production/cd-certs"),
    )?)
    .build()
    .await?;
let fabric_id = controller.create_fabric(FabricConfig { /* fabric_id, rcac_id, commissioner_node_id: 1, validity */ })?;
let node_id = controller.commission("<manual-pairing-code>").await?;
```

## Address caveat (from M7.5)
A manual pairing code carries only the **short** (4-bit) discriminator, so mDNS
commissionable discovery may not match. M6.6's `commission()` discovers via the
discriminator; if discovery fails on your LAN, the device's operational record
still resolves post-AddNOC. If the commissionable phase cannot find the device,
re-run near the device / confirm it is in commissioning mode (blinking), or fall
back to the `commission_ip` example's `--addr` path to confirm reachability first.

## Steps
1. Factory-reset the P110M; confirm it is advertising `_matterc._udp`.
2. Run the commissioning flow (controller built as above) with the device's
   manual pairing code (printed on the device / in the Tapo app).
3. **Expect:** `commission()` returns a non-zero `node_id`; the snapshot file now
   contains one fabric + one `DeviceEntry` whose `peer_noc_public_key` is the
   device's real NOC key (not the `[0u8;65]` placeholder).
4. Control it: `controller.node(node_id).round_trip(0x02 /*ReadRequest*/, ProtocolId::INTERACTION_MODEL, build_read_request(&[OnOff path]))` returns the device's OnOff `ReportData` — i.e. the persisted identity authenticates an operational CASE session.
5. **Restart test:** drop the controller, re-`open` (or re-`build`) it against the
   same store, and repeat step 4 **without** re-commissioning. The actor
   re-establishes CASE from the persisted fabric + commissioner identity and the
   read succeeds. This is the headline M8 property: commission once, persist,
   reconnect from snapshot.

## Recording the result
Note the device, date, the returned `node_id`, and that steps 3–5 succeeded.
Update `docs/tested-devices.md` if present. Commissioning correctness at the
byte level is already covered by `matter-commissioning`'s `commission_loopback`
(hermetic, CI-gated); this runbook validates the **controller integration** on
real hardware.
